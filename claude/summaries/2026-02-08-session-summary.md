# セッション要約: 前提文書レビューと提案書骨格の策定

## 確定した事項

### 提案書の骨格

1. **問題設定**: ソフトウェア工学は行為者（計算機械、アルゴリズム、ひと、接地先）間の会話という問題に取り組んできたが、古典的SEも後アジャイルSEも解決できていない。特に接地先は常に対話の外に置かれてきた。
2. **新たな条件**: 言語モデルは、すべての行為者がそれぞれ持つ道具として位置づけられる。行為者間の翻訳コストを下げ、接地先に「声」を与える可能性を開く。
3. **目指すこと**: 接地先を含むすべての行為者が対話に参加できるソフトウェア工学の構想と、その具体的な手法・枠組みの探求。
4. **成果物・進め方**: 未定（次回以降）
5. **体制・読者へのメッセージ**: 未定（次回以降）

### 重要な概念的決定

- **接地先は行為者である**: 記述される対象ではなく、対話に能動的に参加する（すべき）もの。これが深ソフトウェア工学の核心。
- **言語モデルは道具である**: 第五の行為者でも中央の翻訳装置でもなく、すべての行為者がそれぞれ持つ道具。分散的で、行為者に従属する。ただし、この認識は今後変わりうる。
- **「接地先」の用語はそのまま使う**: 初出時に定義を添える。「記号としての言語が結びつく先にある、物理的・社会的・身体的・生命的な現実」

### 提案書の方向性に関する決定

- **想定読者**: 対等で多様な思考家・実践家のコミュニティ。提案書は招待状のトーン。
- **「深」の意味**: deep learning への掛け言葉 + 「シン・○○」文化への皮肉。提案書では軽く触れる程度。
- **5年後の成果物像**: コミュニティ + 知識（書籍等）+ 実践例（awe9 等）
- **awe9 は前面に出さない**: 読者の混乱を避ける。深SEは awe9 なしで自立した問題設定を持つ。
- **行為者工学には触れない**: 自然に拡がっていくもの。
