# 深ソフトウェア工学 (Deep Software Engineering)

— すべての行為者が対話に参加するソフトウェア工学に向けて —

[English version](en.html)

## はじめに

この文書は, 深ソフトウェア工学 (Deep Software Engineering) についての表明 (statement) です.

わたしたちは, ソフトウェア工学が半世紀にわたって取り組んできた問題を, 言語モデルという新たな道具を手にしたいま, もう一度深く掘り下げる必要があると考えています. この文書は, その考えを表明するものです.

## ソフトウェア工学が発見した問題

ソフトウェア工学は, その始まりから一貫して, ある本質的な問題に取り組んできました.

ソフトウェアに関わる**行為者**は複数存在し, 互いに共通の言語を持ちません.

- **計算機械** — ハードウェア, 物理的な計算基盤
- **アルゴリズム** — ソフトウェア, 計算の手続き
- **ひと** — 開発者, 利用者, その他の関係者
- **接地先** — 記号としての言語が結びつく先にある, 物理的・社会的・身体的・生命的な現実

これらの行為者の間で会話ができるようにすること. ソフトウェア工学の根底にある問題は, 突き詰めればこの一点に収束します.

### 古典的ソフトウェア工学の試み

1970 年代から 2000 年代にかけて発展した古典的ソフトウェア工学は, この問題を正しく発見しました. しかし, それを解決するために採った手段 — 膨大な文書, 硬直的なプロセス, 共通言語の人為的な定義 — は, 行為者間の会話を促進するどころか, むしろ抑圧するものになりました.

文書の作成と検証に膨大なコストがかかり, それを抑えるために硬直的なプロセスが推奨され, ひとの認知限界を超える言語使用が強制される. 根拠の薄い指標が採用され, それに基づく誤った最適化が行われる. 古典的ソフトウェア工学が採った手法が, 最終的には自らの行き詰まりを招きました.

### 後アジャイル・ソフトウェア工学の展開

1990 年代半ばから現在に至る後アジャイル・ソフトウェア工学は, 古典的ソフトウェア工学の硬直性を乗り越えようとしました. フィードバック, 応答性, 分散性, 並行性を重視し, 行為者の種類と行為者間で用いられる言語の種類を増やしました.

わたしたちはここで「言語」という語を広い意味で用いています. パタン言語, グラフィカルなモデル, フレームワーク, 形式的言語と型システム, 領域固有言語, スクリプト, メソドロジ, さらにはコード・カード・ワードといった実践上の記述形式 — これらすべてが, 行為者間の会話を媒介する「言語」です.

この多声的な展開は望ましいものでした. しかし, 初期のアジャイルやリーンが重視していた創造性, 個人の多様性, 批判的思考は, やがて機械的な反復, 画一的なフレームワークの無批判な採用, グローバル資本主義の巨大プラットフォームへの奉仕に置き換わっていきました.

そして, 古典的ソフトウェア工学が行き詰まったのと並行的に, 同じ状況に陥りつつあるように見受けられます.

### 置き去りにされた行為者

古典的ソフトウェア工学も, 後アジャイル・ソフトウェア工学も, ある重大な問題を共有しています.

**接地先が, 常に対話の外に置かれてきた**ということです.

古典的ソフトウェア工学では, 現実は要件定義書に写し取られる対象でした. 後アジャイル・ソフトウェア工学では, 現実はユーザストーリーやモデルに断片的に記述される対象です. いずれにおいても, 接地先は「記述される側」であり, 「表象の客体」であって, 対話に参加する行為者ではありませんでした.

しかし, 接地先は受動的な対象ではありません. 接地先は —

- 制約を課します (物理法則, 社会的規範, 身体の限界)
- フィードバックを返します (期待と異なる振る舞い, 予測不能な変化)
- 自ら変化します (ソフトウェアの介入によって接地先自体が変わる)

接地先は対話の対象であると同時に, 対話に参加している — あるいは, 参加すべきものです.

## 新たな条件

この状況に, いま, 大きな変化が訪れつつあります. 言語モデルの急速な普及です.

わたしたちはここで「生成 AI」のような曖昧な用語を避けて「言語モデル」という語を使います. また「大規模」を付けません. 規模の大小は, 今後本質ではなくなると予想するからです.

言語モデルの良し悪しに関わらず, その影響を見過ごすことはできません. そして, 言語モデルは, ソフトウェア工学が半世紀にわたって取り組んできた問題 — 行為者間の会話 — に対して, これまでにない可能性を開きます.

### 言語モデルは道具である

わたしたちは言語モデルを, 第五の行為者とも, 中央集権的な翻訳装置とも位置づけません.

**言語モデルは, すべての行為者がそれぞれ持つ道具です.**

この認識には重要な含意があります.

- **分散的である**: 中央に一つの翻訳者がいるのではなく, 各行為者がそれぞれの言語モデルを通じて他の行為者と対話する
- **行為者に従属する**: 道具は行為者の意図や文脈に沿って使われる. 言語モデルが対話を支配するのではなく, 行為者が対話を主導する
- **行為者ごとに異なりうる**: ひとが持つ言語モデルと, アルゴリズムが持つ言語モデルは, 同じものである必要がない

### 何が可能になるか

この道具によって, 以下のようなことが可能になると考えます.

- 文書の生成と変更が, 低いコストで, ある程度まで整合的に行われうる
- 文書の検証が, 言語モデルと (半) 形式的手法による記述との突き合わせによって, ある程度まで可能になる
- 自然言語と, ひと以外の行為者が用いる言語との翻訳/通訳が, ある程度まで可能になる
- 行為者間で用いられる言語が, 全行為者が了解するための中途半端なものから, 分野や行為者に最適化されたものに変わりうる
- そして何より, **接地先が対話に参加するための声を得る可能性が開かれる** — センサデータ, 物理的制約, 社会的フィードバックなどが, 言語モデルを通じて対話可能な形になる

古典的ソフトウェア工学が発見した問題に, いま, 異なる手段で取り組むことができる条件が整いつつあるのです.

## 深ソフトウェア工学が目指すこと

わたしたちは, **接地先を含むすべての行為者が対話に参加できるソフトウェア工学**を構想し, その具体的な手法と枠組みを探求する必要があると考えています.

具体的には, 以下のような問いに取り組むべきだと考えます.

1. **言語の再設計**: 行為者間で用いられる言語 (パタン言語, モデル, フレームワーク, 型システム, 領域固有言語など) は, 言語モデルの存在のもとでどのように再設計されうるか
2. **接地先の参加**: 接地先を行為者として対話に参加させるための技術的・方法論的な仕組みは何か
3. **共通理解の再構築**: 古典的ソフトウェア工学が文書によって不完全に実現しようとした行為者間の共通理解を, 言語モデルと (半) 形式的手法の併用でどこまで実現できるか
4. **言語の問い直し**: 言語の表現力とは何か. 言語の使われ方は創造的か, それとも慣習的あるいはクリシェ的か. 言語の意味は十分に共有され, 正しく接地しているか. 言語と行為, 言語と思考は適切に繋がっているか

これらの問いは, ソフトウェア工学に閉じたものではありません. しかし, わたしたちはまずソフトウェア工学という具体的な領域から出発します.
