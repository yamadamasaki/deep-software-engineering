# awe9

awe9 は, 以下に述べるような前提を元にソフトウェアがどうあるべきなのか, あるいはソフトウェアはもう存在する余地がないのかを検討し, 実装し, 実験し, 検証するプロジェクトです.

## 状況

世界的にグローバル資本主義が一般的な経済をも巻き込んで崩壊し, 西洋近代的な自由主義, 民主主義が機能しなくなっている.

技術的には 2026 年の状況に比較すべきほどのものではまったくないが, それでもある程度のインフラストラクチャ, 小規模な工業, 流通は存在している.

ここにはいわば一種の空白状態が生まれている. その中でひとびとがよりよく生きていくためにソフトウェアは有効かを検証しつつ, 新たなソフトウェア (あるいはそれに代わるもの) を作り, 使う必要に迫られている.

## 対象となる社会

ひとびとは, 数十人から数百人程度の (地縁血縁からはある程度独立した) さまざまな種類の集団に緩やかに属し (加入も離脱もそれほど厳しくなく, ひとりが複数の集団に属していることも普通である), それらの集団の間にもさまざまな関係に基づくネットワークが形成されている.

集団のメンバは誰もがお互いに見知っており, 集団の行動が全メンバの個人としての判断を超えて暴走することのない, 緩やかな仕組みを育てている.

それらのひとびとや集団が生き (延び) ていくのに必要なソフトウェア, あるいはそれに代わる何かを提供したい.

それらを提供しようとするひとびとも何らかの (多分複数の) 集団を形作っている.

## 対象となる分野

生き (延び) るのに役立つ分野ならば, すべてが対象となる.

従来の区分で言えばエンタプライズ向けシステム, 個人の知識/情報管理ツール, エンタテイメントのためのアプリケーションなどが含まれるだろうが, 前提となる社会が大きく変動してしまった状況においては, ソフトウェア自体もまったく違ったものにならざるを得ないだろう.

それらが統合された一つのシステムになるのか, あるいはたくさんの多様なアプリケーションの集まりになるのか, あるいはソフトウェアとは全く違うものが必要になるのか, ということも含めて考えていく.

## 利用可能な技術

その時点でどのような技術が利用可能であるかを明確に知ることは難しい.

ただし, 想定しているような状況や, そもそもの awe9 の方向性から以下のような技術は前提としないものとする.

- いわゆるクラウド技術 (特に AWS, Google Cloud, Azure のような巨大なもの)
- いわゆる High Performance Computing 技術
- 高速/高信頼性/広域のネットワーク技術 (今よりも低速, 低信頼性, 狭域のネットワークは存在しうるとする)
- 最新のスマートフォンなどの高度なモバイル・デバイスと無線ネットワーク技術

利用可能な技術の候補として, 例えば以下のようなものを検討している.

- IPFS (Inter-Planetary File System), あるいはその制限版
- DFinity/ICP (Internet Computer Protocol), あるいはその制限版
- ATProto (Authenticated Transfer Protocol) など
- ある程度の型システム, あるいは証明支援系などの形式的手法
- 小型計算機の集約クラスタ
- 普通の文房具
